## New Frame Loading Flow

### Frontend Logic (App.jsx):

```javascript
useEffect(() => {
  async function loadFrame() {
    // 1. Get all product image URLs
    const productImageUrls = rootElement?.dataset?.productImageUrls
      .split(',')
      .map(url => url.trim())
    
    // 2. Detect transparent frame
    const detectResponse = await fetch('/api/detect-frame', {
      method: 'POST',
      body: JSON.stringify({
        productId: currentVariantId,
        imageUrls: productImageUrls,
        shop: window.Shopify?.shop
      })
    })
    
    const detectData = await detectResponse.json()
    
    // 3. Generate frame using detected reference
    const generateResponse = await fetch('/api/generate-frame', {
      method: 'POST',
      body: JSON.stringify({
        productId: currentVariantId,
        referenceFrameUrl: detectData.frameImageUrl, // null if not found
        targetImageUrl: productImageUrl
      })
    })
    
    const generateData = await generateResponse.json()
    
    // 4. Display frame (or null for graceful degradation)
    if (generateData.frameUrl) {
      setAutoGeneratedFrameUrl(generateData.frameUrl)
    } else {
      setAutoGeneratedFrameUrl(null) // Customizer works without frame
    }
  }
  
  loadFrame()
}, [productImageUrl, currentVariantId])
```

### Complete User Flow:

```
1. User opens customizer
   ↓
2. Frontend reads data-product-image-urls from Liquid
   ↓
3. Frontend calls /api/detect-frame with all product images
   ↓
4. Backend downloads images and runs find_transparent.py
   ↓
5. Backend returns frameImageUrl (or null)
   ↓
6. Frontend calls /api/generate-frame
   ↓
7. Backend runs apply_phone_mask.py with reference frame
   ↓
8. Backend returns base64 frame data URL
   ↓
9. Frontend displays frame layer
   ↓
10. User customizes phone case with frame overlay ✅
```

### Graceful Degradation Flow:

```
1. Product has no transparent images
   ↓
2. /api/detect-frame returns { found: false, frameImageUrl: null }
   ↓
3. /api/generate-frame returns { frameUrl: null, message: "..." }
   ↓
4. Frontend sets autoGeneratedFrameUrl = null
   ↓
5. Customizer works without frame layer ✅
```

## Key Features

### 1. Automatic Frame Detection
- No more manual `alt="frame"` tagging required
- Python script automatically detects transparent images
- Uses 25% transparency threshold

## Troubleshooting

### Issue: "Cannot find module '~/lib/python-bridge'"
**Solution**: TypeScript path resolution - will work at runtime. Ignore the error.

### Issue: "Python environment not available"
**Solution**: 
1. Check Python 3 is installed: `python3 --version`
2. Install dependencies: `pip install -r phoneLayer/requirements.txt`
3. Test endpoint: `curl http://localhost:3000/api/test-python`

### Issue: "Frame not loading"
**Solution**:
1. Check browser console for errors
2. Verify API endpoints are responding
3. Check database has ProductFrameCache entries
4. Verify product images are accessible

### Issue: "Prisma migration fails"
**Solution**:
1. Check database file exists: `prisma/dev.sqlite`
2. Run: `npx prisma generate`
3. Try: `npx prisma migrate reset` (WARNING: deletes data)
